Composite Key Implementation
What is a Composite Key?

This system uses a composite unique constraint on (item_code, description) instead of a single unique constraint on item_code alone.

Benefits:
  • Allows multiple variants of the same item code (e.g., "1007 - HF Pulled pork - WIP" and "1007 - GB Pulled Pork - WIP")
  • Maintains data integrity by preventing exact duplicates
  • Supports business scenarios where same item code has different variants
  • Preserves existing item coding conventions
Example:
Item Code Description Status
1007 HF Pulled pork - WIP Allowed
1007 GB Pulled Pork - WIP Allowed
1007 HF Pulled pork - WIP Duplicate (Blocked)
